# Intent Understanding Engine

Clean two-tier architecture — no middleware:

```
Browser (React)
      │
      │  POST http://localhost:5001/predict
      ▼
Flask (Python)
      │
      │  TF-IDF vectorise → Logistic Regression
      ▼
{ intent, confidence, entities, action, scores }
```

---

## Project structure

```
intent-system/
├── nlu/
│   ├── data/
│   │   └── intents.json          ← 60 labelled training examples (6 intents)
│   ├── models/
│   │   └── intent_classifier.pkl ← saved model (generated by train.py)
│   ├── train.py                  ← TF-IDF + Logistic Regression trainer
│   ├── predict.py                ← inference + entity extraction
│   ├── app.py                    ← Flask REST API  (port 5001)
│   └── requirements.txt
│
└── ui/
    ├── src/
    │   ├── main.jsx              ← React entry point
    │   └── App.jsx               ← Full UI — calls Flask directly
    ├── index.html
    ├── package.json
    └── vite.config.js
```

## Quick start

### 1 — Python Flask backend  (terminal 1)

```bash
cd nlu
pip install -r requirements.txt

# Train the model (only needed once, or when you add new training data)
python train.py

# Start the API
python app.py
# → http://localhost:5001
```

### 2 — React frontend  (terminal 2)

```bash
cd ui
npm install
npm run dev
# → http://localhost:5173
```

Open **http://localhost:5173** and start typing.

## Extending

### Add a new intent
1. Add examples to `nlu/data/intents.json`
2. Re-run `python train.py`
3. Add the intent to `INTENTS` in `ui/src/App.jsx`

### Add a new entity type
Edit the regex patterns in `nlu/predict.py` → bottom of the file.

### Connect to real services
Swap the action string in `predict.py` → `ACTIONS` dict for real API calls
(Gmail, Google Calendar, Twilio, Slack, etc.).
